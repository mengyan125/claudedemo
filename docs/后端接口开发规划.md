# 学生反馈系统 - 后端接口开发规划

## 一、背景

前端已完成开发（Vue 3 + Element Plus），使用 Mock 数据运行。后端代码**完全空白**，需要从零搭建 Spring Boot 项目并实现全部 API 接口，使前端可以对接真实后端。

前端已定义好完整的 API 接口路径、请求参数类型和响应数据结构（见 `src/frontend/src/api/*.ts` 和 `src/frontend/src/mock/index.ts`），后端实现必须严格匹配这些契约。

---

## 二、技术方案

| 项目 | 选型 | 说明 |
|------|------|------|
| 框架 | Spring Boot 2.7.18 | JDK 1.8 兼容的最新稳定版 |
| ORM | MyBatis-Plus 3.5.3 | 自动 CRUD + 内置分页，减少 XML |
| 数据库 | TiDB 8.5 | 兼容 MySQL 协议，通过 SOCKS 代理连接 |
| 认证 | JWT (jjwt 0.9.1) | 自定义拦截器，不启用 Spring Security 过滤链 |
| 缓存 | 暂不引入 | 后续优化阶段再加 JetCache + Lettuce |
| 密码 | BCryptPasswordEncoder | 仅引入编码器 |
| 校验 | spring-boot-starter-validation | DTO 参数校验 |
| 测试 | spring-boot-starter-test + MySQL | 使用 MySQL 测试库做测试 |

### 关键技术决策

1. **Controller 路径前缀**：前端 `baseURL: '/api'`，Vite 代理到 `localhost:8080` 不做路径重写，所以后端 `@RequestMapping` 路径必须带 `/api` 前缀
2. **SOCKS 代理连接 TiDB**：使用 MySQL JDBC 驱动原生参数 `socksProxyHost=127.0.0.1&socksProxyPort=51080`
3. **驼峰映射**：MyBatis-Plus 开启 `map-underscore-to-camel-case: true`，数据库下划线自动转前端 camelCase
4. **单一数据源 profile**：默认使用 `application.yml` 激活 `mysql`，`application-tidb.yml` 仅用于 TiDB 连接（待提供连接信息）
5. **缓存延后**：第1~7阶段不引入 JetCache/Redis，第8阶段优化时再加

## 四、分阶段开发计划

### 第1阶段：项目骨架 + 认证

**目标：** Spring Boot 项目可启动、JWT 认证链路跑通、统一响应/异常处理就绪

**要创建的文件：**

| 文件 | 说明 |
|------|------|
| `pom.xml` | Maven 依赖（Spring Boot 2.7.18、MyBatis-Plus、jjwt、MySQL、Lombok 等） |
| `FeedbackApplication.java` | `@SpringBootApplication` + `@MapperScan` |
| `Result.java` / `PageResult.java` | 统一响应格式 |
| `BusinessException.java` | 业务异常类 |
| `GlobalExceptionHandler.java` | 全局异常处理（BusinessException、参数校验、兜底Exception） |
| `CorsConfig.java` | 允许前端跨域 |
| `MyBatisConfig.java` | 注册 PaginationInnerInterceptor |
| `WebMvcConfig.java` | 注册 JwtInterceptor，排除 `/api/auth/login`、`/api/files/**` |
| `JwtUtil.java` | generateToken / parseToken / getUserId / isTokenExpired |
| `JwtInterceptor.java` | 从 `Authorization: Bearer` 提取 token → 解析 → 存 UserContext |
| `UserContext.java` | ThreadLocal 存储/获取/清理当前用户信息 |
| `SysUser.java` | 用户实体 |
| `SysUserMapper.java` | 继承 `BaseMapper<SysUser>` |
| `AuthService.java` / `AuthServiceImpl.java` | 登录逻辑 |
| `AuthController.java` | `POST /api/auth/login`、`POST /api/auth/logout` |
| `application.yml` / `application-mysql.yml` | 数据源、MyBatis、JWT、文件上传配置 |
| `db/V1__create_sys_tables.sql` | sys_user, sys_role, sys_user_role 建表 |
| `db/V4__init_data.sql` | 默认管理员 admin/123456 + 三个角色 |

**登录响应格式**（匹配前端 `api/auth.ts`）：

```json
{
  "code": 200, "message": "success",
  "data": {
    "token": "eyJ...",
    "userInfo": {
      "id": 1,
      "username": "admin",
      "realName": "系统管理员",
      "userType": "admin",
      "roles": ["SYSTEM_ADMIN"],
      "schoolName": "示范学校"
    }
  }
}
```

**JWT 认证流程：**
1. 登录：查 sys_user → BCrypt 校验密码 → 生成 JWT（payload 含 userId/username/userType） → 查角色列表 → 返回 token + userInfo
2. 请求拦截：JwtInterceptor 解析 Bearer token → 存入 UserContext → 放行。失败返回 401
3. 登出：前端清除 token，后端无需操作
4. afterCompletion 中清理 ThreadLocal 防内存泄漏

| `BaseGradeController` | 4 | `/api/base/grade` | GET /list, POST, PUT /{id}, DELETE /{id} |
| `BaseClassController` | 4 | `/api/base/class` | GET /list?gradeId=, POST, PUT /{id}, DELETE /{id} |
| `BaseRelationController` | 7 | `/api/base/relation` | GET /class/{classId}/students, GET /students/unassigned, POST /class/{classId}/students, DELETE /student-assignment/{id}, GET /class/{classId}/teachers, POST /class/{classId}/teachers, DELETE /teacher-assignment/{id} |

**新增 Entity：** SysRole, SysUserRole, BaseSemester, BaseGrade, BaseClass, BaseStudentClass, BaseTeacherClass

**SQL：** `db/V2__create_base_tables.sql`

**注意点：**
- 用户列表返回的 `roles` 是字符串数组（如 `["SYSTEM_ADMIN"]`），需联查 sys_user_role + sys_role
- 用户列表分页，使用 MyBatis-Plus 的 `Page` 对象，返回 `PageResult<UserVO>`
- 批量创建用户接收 `List<CreateUserDTO>`，每个密码做 BCrypt 加密
- 设置当前学期：先将所有学期 `is_current=false`，再将目标学期 `is_current=true`（事务）
- 班级列表需带 `gradeName`（LEFT JOIN base_grade）
- 未分配学生 = `user_type='student'` 且不在当前学期 base_student_class 中
- 师生关系查询需联表获取 realName 和 username

### 第4阶段：学生端

**目标：** 学生提交/暂存反馈、反馈跟踪、详情、追加回复、文件上传

**Controller 清单（2个，共7个接口）：**

| Controller | 接口数 | 路径前缀 | 接口列表 |
|-----------|-------|---------|---------|
| `StudentFeedbackController` | 6 | `/api/student` | GET /categories, GET /teachers, POST /feedback, GET /feedback/list, GET /feedback/{id}, POST /feedback/{feedbackId}/reply |
| `FileUploadController` | 1 | `/api/upload` | POST /file |

**新增 Entity：** FbFeedback, FbFeedbackAttachment, FbReply

**SQL：** `db/V3__create_feedback_tables.sql`

**关键逻辑：**

| 接口 | 逻辑说明 |
|------|---------|
| GET /categories | 查 fb_category 中 status=1（启用）的记录，返回 {id, name, isTeachingRelated} |
| GET /teachers | 当前学生 → base_student_class(当前学期) → base_teacher_class → sys_user，返回 {id, name, subject} |
| POST /feedback | status='draft'(暂存) 或 'submitted'(提交)；有 id 则更新，无 id 则新建；关联 attachmentIds 到 fb_feedback_attachment |
| GET /feedback/list | 只查当前学生的反馈，支持 categoryId/keyword 筛选，分页返回 PageResult |
| GET /feedback/{id} | 校验归属(student_id=当前用户)，关联附件和回复列表，查看后标记 has_unread_reply=false |
| POST /feedback/{feedbackId}/reply | 校验归属，插入 fb_reply，更新 reply_status |
| POST /upload/file | 校验类型(jpg/png/gif/mp4/avi/mov)和大小(<=25MB)，存本地磁盘 `{uploadDir}/{yyyyMMdd}/{UUID}.{ext}`，先建 attachment 记录(feedback_id=null)，返回 {id, fileName, fileUrl} |

**文件存储：** WebMvcConfig 做静态资源映射 `/api/files/**` → 本地磁盘目录

## 六、前后端接口契约核查清单

| 检查项 | 前端定义位置 | 后端注意点 |
|--------|-------------|-----------|
| 登录返回 roles 数组 | `api/auth.ts` LoginVO | 联查 sys_user_role + sys_role，返回角色 code 数组 |
| 用户列表 roles 字段 | `mock/index.ts` MOCK_USERS | 联查角色，返回字符串数组如 `["SYSTEM_ADMIN"]` |
| 班级带 gradeName | `mock/index.ts` MOCK_CLASSES | LEFT JOIN base_grade 查年级名称 |
| 反馈列表 content 截断 | `mock/index.ts` 行 281 | 列表接口只返回前 30 字摘要 + "……" |
| 管理员详情带 isFavorited | `api/admin.ts` AdminFeedbackDetail | 查 fb_collection 判断当前用户收藏状态 |
| 回复带 replyUserType | `api/admin.ts` AdminReply | 根据 reply_user_id 联查 sys_user.user_type，返回 'admin' 或 'student' |
| 统计 6 个数据块 | `api/admin.ts` FeedbackStatistics | 单接口返回复合对象（totalCount/semesterCount/monthCount/categoryDistribution/semesterTrend/teacherTop10） |
| 收藏接口无请求体 | `api/admin.ts` toggleFavoriteApi | 后端自行判断当前状态并切换 |
| 文件上传返回格式 | `api/student.ts` uploadFileApi | 返回 `{id, fileName, fileUrl}`，先建 attachment 记录 |
| 学生反馈列表 | `api/student.ts` FeedbackListItem | 包含 status 字段，值为 'draft'/'submitted'/'replied' |
| 管理员反馈列表 | `api/admin.ts` AdminFeedbackItem | 不包含 draft 状态的反馈，status 只有 'submitted'/'replied' |

| 层次 | 工具 | 策略 |
