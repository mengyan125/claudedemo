# 学生反馈系统详细测试用例文档

## 1. 文档说明

- 文档目标：提供可直接执行的详细测试用例，覆盖从“空白系统”到“业务闭环”的完整链路。
- 适用环境：
  - 前端：`http://localhost:3000`
  - 后端：`http://localhost:8080`
  - 数据库：MySQL（默认 profile）
  - 前端配置：`VITE_ENABLE_MOCK=false`
- 角色范围：
  - 学生（student）
  - 系统管理员（SYSTEM_ADMIN）
  - 角色管理员（ROLE_ADMIN）
  - 类别管理员（CATEGORY_ADMIN）

---

## 2. 执行顺序（按真实落地）

1. 环境与系统可用性
2. 基础数据维护（学期、年级、班级、师生关系）
3. 权限与组织配置（用户、角色、类别、类别授权）
4. 冒烟测试（登录、路由、拦截）
5. 反馈业务主链路（学生提交、管理员处理、提醒）
6. 统计与下钻
7. 权限与边界
8. 回归最小集

---

## 3. 测试用例清单（详细）

## A. 环境与系统可用性

### TC-A01 前后端服务可用
- 前置条件：服务已启动

  1. 打开 `http://localhost:3000`
  2. 访问后端健康入口（或任意可达接口）
- 预期结果：
  - 前端页面可访问
  - 后端可响应请求

### TC-A02 系统管理员登录
- 前置条件：系统管理员账号可用
- 步骤：
  1. 打开 `/login`

  3. 点击登录

## E. 反馈业务主链路（核心）

## E1 学生端

### TC-E101 新建反馈（非匿名）提交
- 角色：学生
- 步骤：
  1. 打开 `/student/feedback`
  2. 填写类别、教师、标题、内容
  3. 关闭匿名并提交
- 预期结果：
  - 提交成功
  - 在 `/student/tracking` 可见新记录
- 现网规则备注（2026-03-20实测）：
  - 反馈对象会根据反馈类别动态变化
  - 上传附件为非必填项

### TC-E102 新建反馈（匿名）提交
- 角色：学生
- 步骤：
  1. 新建反馈
  2. 勾选匿名并提交
- 预期结果：
  - 提交成功
  - 管理员侧不显示学生真实身份

### TC-E103 草稿保存与二次编辑提交
- 角色：学生
- 步骤：
  1. 新建反馈并保存草稿
  2. 在跟踪页打开草稿继续编辑
  3. 提交草稿
- 预期结果：
  - 草稿可编辑
  - 状态由草稿变为已提交

### TC-E104 学生查看详情与追加回复
- 角色：学生
- 前置条件：该反馈已有管理员回复
- 步骤：
  1. 进入 `/student/tracking/:id`
  2. 追加回复并提交
- 预期结果：
  - 追加成功
  - 回复记录即时可见

### TC-E105 我要反馈空白提交校验
- 角色：学生
- 步骤：
  1. 进入 `/student/feedback`
  2. 不填写任何必填项直接提交
- 预期结果：
  - 前端给出必填校验提示
  - 不发起提交请求

### TC-E106 我要反馈部分字段提交校验
- 角色：学生
- 步骤：
  1. 仅填写标题或仅填写内容后提交
- 预期结果：
  - 缺失项有明确提示
  - 提交失败

### TC-E107 类别切换与反馈对象联动
- 角色：学生
- 步骤：
  1. 选择类别A并观察反馈对象列表
  2. 切换到类别B
- 预期结果：
  - 反馈对象随类别动态变化
  - 无效对象会被清空或要求重选

## E3 状态流转（已读/未读、已回复/未回复）

### TC-E301 学生端已读/未读状态变化
- 角色：学生
- 前置条件：存在管理员新回复的反馈
- 步骤：
  1. 学生进入反馈跟踪列表，记录目标反馈初始状态
  2. 打开反馈详情
  3. 返回列表并刷新
- 预期结果：
  - 打开详情后，状态从未读变为已读（若系统有该标识）
  - 状态展示与详情阅读行为一致

### TC-E302 管理端已读/未读状态变化
- 角色：管理员
- 前置条件：学生新增或追加回复后，管理员尚未查看
- 步骤：
  1. 管理员在反馈列表查看目标反馈状态
  2. 打开反馈详情并返回列表
  3. 刷新页面
- 预期结果：
  - 打开详情前后，未读/已读状态正确变化
  - 列表与详情状态一致

### TC-E303 管理端已回复/未回复状态变化
- 角色：管理员
- 步骤：
  1. 新提交一条未回复反馈
  2. 在列表确认“未回复”状态
  3. 进入详情发送回复
  4. 返回列表刷新
- 预期结果：
  - 回复前为未回复
  - 回复后变为已回复
  - 筛选“已回复/未回复”结果准确

### TC-E304 学生端回复后管理员端未读标识
- 角色：学生 + 管理员
- 步骤：
  1. 学生对已处理反馈追加回复
  2. 管理员查看对应反馈列表状态
  3. 管理员打开详情后再回列表
- 预期结果：
  - 学生新增回复后，管理员侧出现未读/待处理提示
  - 管理员查看后状态恢复为已读/已处理（按系统定义）

### TC-E305 提醒消息已读/未读状态
- 角色：管理员
- 前置条件：存在一条新提醒
- 步骤：
  1. 进入提醒列表查看提醒状态
  2. 打开提醒对应详情
  3. 返回提醒列表刷新
- 预期结果：
  - 新提醒初始为未读
  - 查看后变为已读
  - 状态切换与提醒数量统计一致
