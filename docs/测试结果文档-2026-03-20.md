## 4. 规则确认项（本轮新增）

以下为本轮测试中确认到的“现有行为”，暂不判定为缺陷：

1. 年级页面数据不按学期隔离，所有学期可见。
2. 班级页面数据不按学期隔离，所有学期可见。
3. 师生关系页面未选班级时默认展示第一个班级。
4. 师生关系分配教师时“学科”为必填项。
5. 学生“我要反馈”页面中，反馈对象会根据反馈类别动态显示。
6. “上传附件”为非必填。
7. ROLE_ADMIN 默认可见所有反馈类别数据。

## 6. 权限边界专项补测结果（追加）

### 6.1 学生越权访问他人反馈详情
- 结果：通过（业务拦截生效）
- 现象：页面提示“无权查看该反馈”
- 返回：HTTP 200，业务 `code=500`，`message=无权查看该反馈`
- 备注：建议后续优化为 HTTP 403 语义化返回（非阻断）

### 6.2 学生访问管理员页面
- 结果：通过（前端路由拦截生效）
- 现象：访问 `/admin/feedback` 后直接跳回 `http://localhost:3000/student/feedback`
- 说明：未触发 `/api/admin/feedback/list` 请求，拦截发生在请求前

### 6.3 CATEGORY_ADMIN 数据范围
- 结果：通过（按当前设计）
- 现象：类别管理员仅展示授权类别数据
- 说明：类别筛选功能对该角色隐藏，避免主动跨类别查询
