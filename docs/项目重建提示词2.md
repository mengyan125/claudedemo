# 学生反馈系统 - 项目重建提示词

请从零创建一个"学生反馈系统"全栈项目。技术栈：前端 Vue 3.5 + TypeScript + Vite 6 + Element Plus 2.9（中文）+ Pinia 3 + Axios，使用 pnpm 管理；后端 Spring Boot 2.7.18 + JDK 1.8 + MyBatis-Plus 3.5.3.1 + MySQL 5.7（数据库名 feedbackdb，root/123456）+ JWT 认证（jjwt 0.9.1）+ BCrypt 密码加密（仅引 spring-security-crypto，不用完整 Spring Security）。前端端口 3000，后端端口 8080，Vite 代理 /api → localhost:8080。

**数据库设计（16张表）**：系统模块 sys_user（含 username/password/real_name/user_type[student/teacher/admin]/status/school_name/deleted 逻辑删除）、sys_role（固定三个角色：SYSTEM_ADMIN/ROLE_ADMIN/CATEGORY_ADMIN）、sys_user_role；基础数据模块 base_semester（含 is_current）、base_grade、base_class（关联 grade_id）、base_student_class（关联 student_id/class_id/semester_id）、base_teacher_class（多一个 subject 字段）；反馈业务模块 fb_category（含 is_teaching_related/status/sort_order）、fb_category_admin（类别-管理员关联）、fb_feedback（含 category_id/student_id/teacher_id/title/content/is_anonymous/status[draft/submitted/replied]/reply_status[unreplied/replied]/has_unread_reply/has_unread_for_admin）、fb_feedback_attachment（feedback_id 上传时可为 null）、fb_reply、fb_collection（收藏）、fb_reminder + fb_reminder_receiver（备注提醒）、fb_quick_reply（含 is_active）、fb_feedback_admin_read（管理员已读记录，联合唯一键）。初始数据包括：admin/teacher1-3/student1-2 六个用户（密码均 123456 BCrypt），3 个角色，admin 分配 SYSTEM_ADMIN，1 个当前学期，高一高二高三 3 个年级，3 个班级，师生关系数据，3 个反馈类别（教学质量/校园管理/后勤服务）。

**权限体系**：双层模型 userType + role。JwtInterceptor 验证 token 并加载用户角色存入 ThreadLocal（UserContext），RoleInterceptor 读取自定义 @RequiresRole 注解校验权限。SYSTEM_ADMIN 拥有全部功能，ROLE_ADMIN 拥有反馈/统计/类别/用户管理，CATEGORY_ADMIN 仅能查看被分配类别的反馈（Service 层自动过滤 category_id）。/api/auth/login 免认证，/api/student/** 限 student，/api/admin/** 和 /api/system/** 按角色控制。

**共 56 个 API 接口**：认证 3 个（login/me/logout）；学生端 7 个（获取类别、获取任课教师、提交/暂存反馈、反馈列表、反馈详情含已读标记、追加回复、文件上传）；管理员反馈 9 个（列表含 8 项筛选、状态统计、详情、回复、收藏切换、删除、备注提醒、提醒列表、用户搜索）；类别管理 5 个（CRUD + 启停用）；快捷回复 5 个（CRUD + 设置启用）；统计 3 个（概览含类别饼图/学期趋势/Top10、教师 Top10、教师详情）；用户管理 6 个（CRUD + 批量创建 + 启停用）；基础数据 20 个（学期 7 个含设置当前学期、年级 4 个、班级 4 个、师生关系 7 个含未分配学生查询）；权限设置 6 个（角色列表、角色用户、分配/撤销角色、类别权限列表/更新）。统一响应格式 `{code:200, message:"success", data:{}}` 和分页格式 `{list, total, pageNum, pageSize}`。

**前端路由与页面**：登录页（左侧品牌+右侧表单，主题色 #2AABCB），MainLayout 顶部导航+角色动态 Tab，学生端 2 个主页面（我要反馈表单含附件上传最多 9 个/反馈跟踪列表含未读角标和加载更多），管理员端（反馈查看三 Tab[全部/收藏/提醒]+8 项筛选、反馈详情含回复和备注提醒两个 Tab、统计页含数字卡片+饼图+趋势图+Top10、类别管理），系统管理子 Tab 导航（用户管理含 Excel 批量导入、学期/年级/班级 CRUD、师生关系左树右列表、角色权限左卡片右用户列表、类别权限配置、快捷回复管理），无权限提示页。路由守卫：无 token 跳登录，刷新时 GET /me 恢复，按 userType 和 roles 跳转。Pinia user store 管理 token（localStorage 持久化）和 userInfo，Axios 封装含 401 自动登出。支持 Mock 数据（VITE_ENABLE_MOCK 环境变量控制）。

**后端规范**：Controller 路径统一 /api 前缀，MyBatis-Plus 驼峰映射+逻辑删除+分页插件，文件上传到 ./uploads 并配置静态资源映射 /api/files/**，CORS 全局配置，全局异常处理（BusinessException/参数校验/唯一键冲突/通用异常）。反馈状态流转：draft→submitted→replied，收藏切换为 toggle 逻辑，文件上传先创建 attachment（feedback_id=null）再在提交反馈时关联。

请按以上规范完整生成所有前后端代码、SQL 脚本和配置文件，确保接口契约匹配、页面功能可用。分 8 个阶段开发：①项目骨架+认证+登录 ②用户+基础数据管理 ③类别+快捷回复 ④学生端 ⑤管理员端 ⑥统计 ⑦权限设置 ⑧联调优化。
