# 学生反馈系统测试流程文档

## 1. 目标

通过手工测试 + 接口联调方式，验证系统核心业务可用性、权限正确性与回归稳定性。

## 2. 角色范围

- 学生（student）
- 系统管理员（SYSTEM_ADMIN）
- 角色管理员（ROLE_ADMIN）
- 类别管理员（CATEGORY_ADMIN）

## 3. 测试环境

- 前端：`http://localhost:3000`
- 后端：`http://localhost:8080`
- 数据库：MySQL（默认 profile）
- 前端配置：`VITE_ENABLE_MOCK=false`

## 4. 测试执行总流程

1. 准备阶段
2. 冒烟测试
3. 学生端闭环测试
4. 管理员端闭环测试
5. 权限与边界测试
6. 回归测试

---

## 5. 详细流程

### 5.1 准备阶段

- 启动前后端服务（MySQL链路）
- 准备四类角色账号
- 打开浏览器 Network 面板与后端日志窗口

**通过标准：**
- 登录接口可用
- 页面可访问
- 请求能正常转发到后端

### 5.2 冒烟测试

- 登录/登出
- 未登录访问受保护页面跳转登录页
- 学生访问管理员页被拦截
- 管理员访问关键页面可打开：
  - `/admin/feedback`
  - `/admin/feedback/:id`
  - `/admin/statistics`

**通过标准：**
- 无页面空白、无重定向死循环、无阻断性错误

### 5.3 学生端闭环测试

1. 新建反馈（匿名/非匿名）
2. 保存草稿并再次编辑
3. 提交反馈
4. 在跟踪页查看列表与详情
5. 追加回复（如适用）

**通过标准：**
- 草稿与提交状态流转正确
- 列表和详情数据一致
- 追加回复成功并可见

### 5.4 管理员端闭环测试

1. 列表筛选（类别、状态、关键词、时间）
2. 进入反馈详情并回复
3. 收藏/取消收藏
4. 发送备注提醒并在提醒区域验证展示
5. （系统管理员）查看统计并进入教师下钻页

**通过标准：**
- 筛选生效、计数合理
- 回复后学生侧状态变化正确
- 收藏、提醒状态即时更新
- 统计与下钻链路可用

### 5.5 权限与边界测试

- 类别管理员仅可见授权类别
- 学生不可访问他人反馈
- 测试边界输入：空关键词、非法时间区间、异常文本

**通过标准：**
- 无越权访问
- 异常输入不导致系统阻断

### 5.6 回归测试

每次修复后至少回归以下链路：

1. 登录鉴权
2. 学生提交 → 管理员处理 → 学生查看
3. 管理员筛选/回复/收藏/提醒
4. 统计页与教师下钻

**通过标准：**
- 四条链路全部通过

---

## 6. 失败定位流程

按以下顺序排查：

1. **前端层**：检查 Network 请求参数、响应结构、状态码
2. **后端层**：检查日志异常（权限、业务、SQL）
3. **数据库层**：核对关键表数据一致性

关键表：
- `fb_feedback`
- `fb_reply`
- `fb_collection`
- `fb_reminder`
- `fb_reminder_receiver`
- `fb_feedback_admin_read`

## 7. 缺陷记录要求（最小集）

每个缺陷至少记录：

1. 标题（角色 + 页面 + 现象）
2. 复现步骤（3~5步）
3. 实际结果
4. 期望结果
5. 证据（截图 + 请求/响应 + 日志关键行）
6. 初判归属（前端/后端/数据库）
7. 优先级（P0/P1/P2）

## 8. 验收标准

- 核心流程全部跑通至少 1 次
- 权限隔离正确，无越权
- 高风险场景稳定通过
- 回归链路稳定通过
