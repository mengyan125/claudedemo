## 提示词正文

请从零开始创建一个完整的"学生反馈系统"全栈项目，包含前端和后端。以下是完整的需求和技术规范。

---

## 一、项目背景与目标

加强教学日常管理，建立学生与学校管理层之间的沟通渠道。核心特性：
- 学生可随时对教学、管理等方面提出意见和建议
- 支持匿名反馈，保护学生隐私
- 反馈信息只有特定管理员可查看，教师无法直接看到
- 权限分级：不同类别的反馈由不同管理员负责
- 为学校管理决策提供数据支持（统计图表）

## 五、角色权限体系

### 5.1 双层权限模型：userType + role

| userType | 说明 |
|----------|------|
| student | 学生，仅访问学生端功能 |
| teacher | 教师，可通过分配角色获得管理权限 |
| admin | 管理员，通过角色控制具体权限 |

| 角色 role_code | 权限范围 |
|---------------|---------|
| SYSTEM_ADMIN | 全部功能（反馈查看/统计/类别管理/快捷回复/用户管理/学期/年级/班级/师生关系/角色/类别权限） |
| ROLE_ADMIN | 反馈查看/统计/类别管理/快捷回复/类别权限/用户管理 |
| CATEGORY_ADMIN | 仅被分配类别的反馈查看/回复 + 快捷回复 |

### 5.2 后端鉴权方案

采用自定义注解 `@RequiresRole` + 两层拦截器：

```
请求 → JwtInterceptor（认证+加载角色） → RoleInterceptor（角色校验） → Controller
```

- **JwtInterceptor**：验证 Bearer token → 解析 userId/username/userType → 查询 sys_user_role + sys_role 加载角色列表 → 存入 ThreadLocal（UserContext）
- **RoleInterceptor**：读取 @RequiresRole 注解 → 检查 userType 或 roles 是否匹配 → 不匹配返回 403
- **afterCompletion**：清理 ThreadLocal 防内存泄漏

### 5.3 接口权限配置

| 接口路径 | 权限 |
|---------|------|
| POST /api/auth/login | 免认证 |
| GET /api/auth/me, POST /api/auth/logout | 仅需认证 |
| /api/student/** | userType = student |
| /api/admin/feedback/** | SYSTEM_ADMIN / ROLE_ADMIN / CATEGORY_ADMIN |
| /api/admin/category/** | SYSTEM_ADMIN / ROLE_ADMIN |
| /api/admin/quick-reply/** | SYSTEM_ADMIN / ROLE_ADMIN / CATEGORY_ADMIN |
| /api/admin/statistics/** | SYSTEM_ADMIN |
| /api/system/user/** | SYSTEM_ADMIN / ROLE_ADMIN |
| /api/system/role/** | SYSTEM_ADMIN |
| /api/system/category-permission/** | SYSTEM_ADMIN / ROLE_ADMIN |
| /api/base/** (写操作) | SYSTEM_ADMIN |
| /api/base/** (读操作) | 仅需认证 |
| POST /api/upload/file | 仅需认证 |
| /api/files/** | 免认证（静态资源） |

### 5.4 CATEGORY_ADMIN 数据过滤

在 Service 层自动添加 `WHERE category_id IN (被分配的类别ID列表)` 条件。SYSTEM_ADMIN 和 ROLE_ADMIN 不受此限制。

## 十一、关键业务逻辑

### 11.1 反馈状态流转

```
draft（草稿）→ submitted（已提交）→ replied（已回复）
```

- 学生暂存 → status='draft'
- 学生提交 → status='submitted'
- 管理员回复 → reply_status='replied', has_unread_reply=1
- 学生查看详情 → has_unread_reply=0
- 管理员查看详情 → 插入 fb_feedback_admin_read 记录

### 11.2 收藏切换

POST 无请求体，后端查 fb_collection：
- 存在 → 删除（取消收藏）
- 不存在 → 插入（收藏）

### 11.3 备注提醒

接收 `{content, receiverIds[]}`：
- 插入 fb_reminder 记录
- 批量插入 fb_reminder_receiver（每个接收人一条）

### 11.4 文件上传流程

1. 前端选择文件 → POST /api/upload/file（multipart/form-data）
2. 后端校验类型/大小 → 保存到磁盘 → 创建 fb_feedback_attachment（feedback_id=null）
3. 返回 {id, fileName, fileUrl}
4. 学生提交反馈时传 attachmentIds[] → 后端更新 attachment 的 feedback_id

### 11.5 学生获取教师列表

当前学生 → base_student_class(当前学期) → base_teacher_class → sys_user，返回 {id, name, subject}

### 11.6 统计聚合查询

使用 MyBatis-Plus 原生 SQL 或 wrapper 实现：
- 总反馈数、本学期数、本月数
- 类别分布（GROUP BY category_id）
- 学期趋势
- 教师 Top10（GROUP BY teacher_id ORDER BY count DESC LIMIT 10）
