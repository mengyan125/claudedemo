# 学生反馈系统 - 实现方案

## 背景

基于已确认的需求规划文档（docs/需求规划文档.md），制定项目具体实现方案。本系统独立维护所有基础数据（学期、年级、班级、师生关系），独立登录，附件本地存储。

---

## 一、技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端框架 | Vue 3 + Vite | Composition API + TypeScript |
| UI 组件库 | Element Plus | 表单、表格、弹窗等 |
| 状态管理 | Pinia | 用户信息、权限、全局状态 |
| 图表 | ECharts | 反馈统计图表 |
| HTTP 请求 | Axios | 封装请求/响应拦截器 |
| 路由 | Vue Router 4 | 路由守卫做权限控制 |
| 后端框架 | Spring Boot 2.x + JDK 1.8 | RESTful API |
| 数据库 | TiDB 8.5 | 兼容 MySQL 协议 |
| 缓存 | JetCache + Lettuce | Redis 缓存 |
| 认证 | JWT | 无状态认证 |
| 文件存储 | 本地磁盘 | 附件上传存储 |
| 测试 | Jest + Playwright | 单元测试 + E2E 测试 |

## 三、数据库设计（核心表）

### 基础数据表

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| sys_user | 用户表 | id, username, password, real_name, user_type(学生/教师/管理员), status |
| sys_role | 角色表 | id, role_name, role_code(SYSTEM_ADMIN/ROLE_ADMIN/CATEGORY_ADMIN) |
| sys_user_role | 用户角色关联 | user_id, role_id |
| base_semester | 学期表 | id, semester_name, start_date, end_date, is_current |
| base_grade | 年级表 | id, grade_name, sort_order |
| base_class | 班级表 | id, class_name, grade_id, sort_order |
| base_student_class | 学生班级关联 | student_id, class_id, semester_id |
| base_teacher_class | 教师任课关联 | teacher_id, class_id, semester_id, subject |

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| fb_category | 反馈类别 | id, name, is_teaching_related, sort_order, status(启用/停用) |

| fb_feedback | 反馈主表 | id, category_id, student_id, teacher_id, subject, content, is_anonymous, status(暂存/已提交), reply_status(未回复/已回复), has_unread_reply |

| fb_reply | 反馈回复 | id, feedback_id, reply_user_id, content, create_time |
| fb_collection | 收藏表 | id, user_id, feedback_id |
| fb_reminder | 备注提醒 | id, feedback_id, sender_id, content, create_time |
| fb_reminder_receiver | 提醒接收人 | reminder_id, receiver_id, is_read |
| fb_quick_reply | 快捷回复模板 | id, content, create_user_id, sort_order |

## 四、API 设计（核心接口）

### 认证
- `POST /api/auth/login` — 登录，返回 JWT token
- `POST /api/auth/logout` — 登出

### 学生端
- `GET /api/student/categories` — 获取可用反馈类别
- `GET /api/student/teachers` — 获取本学期任课教师
- `POST /api/student/feedback` — 提交/暂存反馈
- `GET /api/student/feedback/list` — 反馈跟踪列表（支持筛选、分页加载更多）
- `GET /api/student/feedback/{id}` — 反馈详情
- `POST /api/student/feedback/{id}/reply` — 学生追加回复
- `POST /api/upload/file` — 文件上传

### 管理员端
- `GET /api/admin/feedback/list` — 反馈列表（支持8项筛选）
- `GET /api/admin/feedback/{id}` — 反馈详情
- `POST /api/admin/feedback/{id}/reply` — 回复反馈
- `POST /api/admin/feedback/{id}/collect` — 收藏/取消收藏
- `POST /api/admin/feedback/{id}/reminder` — 发送备注提醒
- `GET /api/admin/reminder/list` — 提醒关注列表
- `GET /api/admin/collection/list` — 我的收藏列表
- `GET /api/admin/statistics/overview` — 统计概览（本日/本月/类别占比等）
- `GET /api/admin/statistics/teacher-top10` — 教师被反馈Top10
- `GET /api/admin/statistics/teacher/{id}/detail` — 教师被反馈详情列表
- `GET /api/admin/user/search` — 搜索提醒人列表（备注提醒时选择人员）

### 反馈管理
- `GET /api/admin/category/list` — 类别列表
- `POST /api/admin/category` — 新增类别
- `PUT /api/admin/category/{id}` — 编辑类别
- `DELETE /api/admin/category/{id}` — 删除类别
- `PUT /api/admin/category/{id}/status` — 启用/停用
- `GET /api/admin/quick-reply/list` — 快捷回复列表
- `POST /api/admin/quick-reply` — 新增快捷回复
- `PUT /api/admin/quick-reply/{id}` — 编辑
- `DELETE /api/admin/quick-reply/{id}` — 删除

### 系统管理
- `GET /api/system/role/list` — 角色列表
- `POST /api/system/user` — 创建用户
- `POST /api/system/user/batch` — 批量创建用户
- `GET /api/system/category-permission/list` — 类别权限列表
- `POST /api/system/category-permission` — 分配权限
- `DELETE /api/system/category-permission` — 移除权限

### 基础数据
- `GET /api/base/grade/list` — 年级列表
- `GET /api/base/class/list?gradeId=` — 班级列表（联动年级）
- `GET /api/base/semester/current` — 当前学期

## 六、开发阶段规划

| 阶段 | 内容 | 依赖 |
|------|------|------|
| 第1阶段 | 项目骨架搭建（前后端项目初始化、基础配置、JWT认证、通用组件） | 无 |
| 第2阶段 | 基础数据模块（用户、角色、学期、年级班级、师生关系） | 第1阶段 |
| 第3阶段 | 反馈管理模块（类别管理、快捷回复管理） | 第2阶段 |
| 第4阶段 | 学生端（我要反馈、暂存、编辑暂存、反馈跟踪、详情） | 第2、3阶段 |
| 第5阶段 | 管理员端（反馈查看、详情回复、备注提醒、收藏） | 第2、3阶段 |
| 第6阶段 | 反馈统计（图表、Top10、导出） | 第4、5阶段 |
| 第7阶段 | 权限设置（角色管理、类别权限配置） | 第2阶段 |
| 第8阶段 | 联调、测试、优化 | 全部 |

## 七、验证方式

- 每个阶段完成后运行 `npm test` 执行单元测试
- 后端 TDD：先写测试用例再实现功能
- Playwright E2E 测试覆盖核心流程（提交反馈→管理员回复→学生查看）
- 接口测试覆盖率 > 95%
